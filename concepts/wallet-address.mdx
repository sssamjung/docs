# 지갑 주소 체계

Pulse 네트워크는 기존 이더리움 42자 문자열 주소 체계를 기반으로, 참여자 유형에 따라 커스터마이징된 주소 체계를 사용합니다.

## 주소 체계 개요

Pulse 네트워크의 지갑 주소는 **keccak256 해시 알고리즘**을 사용하여 생성되며, 20byte로 절삭됩니다.

```
0x + 40자리 16진수 = 42자리 문자열
```

### 참여자별 주소 예시

| 참여자 유형 | 주소 예시 |
|------------|----------|
| **개인투자자** | `0xd14f723e859ec4f3edee57bf78ceb7c81de08dfb` |
| **법인투자자** | `0x734c49630caC28EaaC33e9722268e64cA80AbcFd` |
| **발행사 (자기계좌부)** | `0xa13Ff1b6930c25d96baf2165A9f3B83d057D2D73` |

## 주소와 계좌의 관계

### 기존 전자증권 체계

```
┌──────────────────────────────────────────────────────────┐
│                      전자등록기관                         │
│                    (한국예탁결제원)                        │
│  ┌────────────────┐  ┌────────────────────────────────┐ │
│  │발행인관리계좌부│  │      고객관리계좌부 (총량)      │ │
│  │    (200)      │  │          (150)                 │ │
│  └────────────────┘  └────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────┐
│                     계좌관리기관                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │A고객계좌부│  │B고객계좌부│  │C고객계좌부│              │
│  │   (30)   │  │   (50)   │  │   (70)   │              │
│  └──────────┘  └──────────┘  └──────────┘              │
└──────────────────────────────────────────────────────────┘
```

## 2-Tier에서 1-Tier로의 개선

### 기존 2-Tier 구조

```
전자등록기관 (예탁결제원)
       │
       ▼
계좌관리기관 (증권사)
       │
       ▼
    투자자
```

### Pulse 1-Tier 구조

```
분산원장 (Pulse Blockchain)
       │
       ├──────────────────────────────┐
       │                              │
       ▼                              ▼
계좌관리기관 (증권사)             전자등록기관 (예탁결제원)
       │                        [모니터링/총량관리]
       ▼
    투자자
```

<Info>
  토큰증권 법제화 이후에는 블록체인이 기존 고객계좌부·자기계좌부 총량관리 방식을 대체합니다. 예탁결제원은 블록체인 네트워크에 접속하여 계좌관리기관이 적절히 기입하고 있는지 확인합니다.
</Info>

## 주소 생성 및 관리

### 계좌관리기관의 역할

1. **투자자 계좌 개설**
   - 증권 계좌 개설 또는 기존 계좌 연동

2. **블록체인 주소 생성**
   - Pulse API를 통해 분산원장 주소값 생성

3. **매핑 관리**
   - 증권 계좌 번호 ↔ 블록체인 주소 매핑

### 주소 생성 API

```javascript
// 투자자 지갑 주소 생성 요청
POST /main/v1/wallet/create
{
  "type": "individual", // "individual" | "corporate" | "issuer"
  "accountId": "123-456-789"
}

// 응답
{
  "address": "0xd14f723e859ec4f3edee57bf78ceb7c81de08dfb",
  "accountId": "123-456-789",
  "createdAt": "2024-01-15T10:30:00Z"
}
```

## 총량관리

### 분산원장 기반 총량관리

Pulse 네트워크는 지갑 주소 체계를 통해 전자등록기관이 총량관리를 수행할 수 있도록 지원합니다:

```
┌──────────────────────────────────────────────────────────┐
│                   Pulse Network                          │
│                                                          │
│                   ┌─────────────┐                       │
│                   │  RPC Node   │◀────── 예탁결제원      │
│                   └──────┬──────┘       (총량관리)       │
│                          │                               │
│     ┌────────────────────┼────────────────────┐         │
│     │                    │                    │         │
│     ▼                    ▼                    ▼         │
│ ┌────────┐          ┌────────┐          ┌────────┐     │
│ │0x734c..│          │0x7ed97.│          │0x13Ff1.│     │
│ │Smart A │          │Smart B │          │Smart C │     │
│ └────────┘          └────────┘          └────────┘     │
└──────────────────────────────────────────────────────────┘
```

### 총량 확인 API


## 주소 유형별 특징

### 개인투자자 주소

| 항목 | 내용 |
|------|------|
| **KYC** | 필수 |
| **이체 한도** | 적용 가능 |
| **개인정보** | 분산원장에 직접 저장 안 함 |
| **계좌 연동** | 1:1 매핑 |

### 법인투자자 주소

| 항목 | 내용 |
|------|------|
| **법인 인증** | 사업자등록번호 확인 |
| **대리인** | 복수 Operator 지정 가능 |
| **감사 추적** | 모든 거래 기록 보관 |

### 발행사 주소 (자기계좌부)

| 항목 | 내용 |
|------|------|
| **역할** | 토큰 최초 발행 수령 |
| **권한** | Issuer Key와 연동 |
| **배정** | 투자자에게 토큰 배정 시 사용 |

## 주소와 KYC

### KYC 등록 프로세스

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│  투자자   │────▶│계좌관리기관  │────▶│ Pulse KYC    │
│ 본인확인 │     │  KYC 수행   │등록  │    API      │
└──────────┘     └──────────────┘     └──────────────┘
```

### KYC 상태와 토큰 전송

<Warning>
  KYC 등록되지 않은 주소로는 토큰 전송이 불가능합니다.
</Warning>

```javascript
// KYC 등록
POST /main/v1/kyc/{token}/{holder}

// KYC 확인
GET /main/v1/kyc/{token}/{holder}
// 응답: { "registered": true }

// 전송 전 KYC 확인 후 진행
if (await checkKYC(toAddress)) {
  await transfer(token, from, to, amount);
} else {
  throw new Error('KYC 미등록 주소');
}
```

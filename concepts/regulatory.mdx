# 금융위 분산원장 요건

2023년 2월 금융위원회는 '토큰증권(Security Token) 발행·유통 규율체계 정비방안'을 발표하여 토큰증권 분산원장이 갖추어야 할 요건을 제시했습니다. Pulse 네트워크는 이 요건을 모두 충족합니다.

## 분산원장 요건 (안)

<AccordionGroup>
  <Accordion title="1. 시간순 기록 및 변경 방지">
    **요건**: 권리자 정보 및 거래 정보가 시간 순서대로 기록되고, 사후적인 조작·변경이 방지될 것

    **Pulse 구현**:
    - 블록체인 특성상 모든 트랜잭션은 타임스탬프와 함께 순차적으로 기록
    - 블록 해시 체인으로 사후 변경 불가능
    - QBFT 합의로 블록 확정 후 변경 불가
  </Accordion>

  <Accordion title="2. 동일성 입증 가능">
    **요건**: 분산원장에 기록된 권리자 정보 및 거래정보와 실제거래내역 사이의 동일성이 계좌관리기관의 책임으로 입증 가능할 것

    **Pulse 구현**:
    - 계좌관리기관이 Operator Key로 모든 거래 서명
    - 트랜잭션 해시로 거래 추적 가능
    - Block Explorer에서 전체 이력 조회 가능
  </Accordion>

  <Accordion title="3. 복수 분산 장부 기록">
    **요건**: 권리자 정보 및 거래 정보가 복수의 분산된 장부에 동일하게 기록될 것

    **Pulse 구현**:
    - 다수의 Validator Node가 동일한 원장 유지
    - QBFT 합의로 모든 노드의 상태 동기화
    - 2/3 이상 노드 합의 시에만 블록 확정
  </Accordion>

  <Accordion title="4. 독립 계좌관리기관 참여">
    **요건**: 전자등록기관, 금융기관 또는 발행인과 특수관계인에 해당하지 않는 계좌관리기관이 다수 참여하여 분산원장을 확인할 수 있을 것

    **Pulse 구현**:
    - 신한투자증권, SK증권 등 독립 금융기관이 노드로 참여
    - 발행인과 특수관계 없는 계좌관리기관 참여
    - RPC Node를 통해 원장 조회 가능
  </Accordion>

  <Accordion title="5. 별도 가상자산 불필요">
    **요건**: 권리자 및 거래정보 기록을 위해 별도의 가상자산을 필요로 하지않을 것

    **Pulse 구현**:
    - 프라이빗 네트워크로 Gas Fee 미부과
    - 채굴 보상 없음
    - 별도 코인/토큰 불필요
  </Accordion>

  <Accordion title="6. 적합한 권리 등록">
    **요건**: 분산원장으로 기록하기 적합한 권리를 등록할 것 (상장증권, 상장 DR, 파생결합증권 제외)

    **Pulse 구현**:
    - 투자계약증권, 수익증권, 채무증권 지원
    - 상장증권, 상장 DR, 파생결합증권 배제
    - ERC-1400 표준으로 적절한 권리만 등록
  </Accordion>

  <Accordion title="7. 법령 준수">
    **요건**: 개인정보보호법, 신용정보법 등 법령을 위반하지 않을 것

    **Pulse 구현**:
    - 개인정보는 분산원장에 직접 저장하지 않음
    - 지갑 주소와 개인정보 분리 관리
    - 계좌관리기관이 개인정보 관리
  </Accordion>

  <Accordion title="8. 금융기관 노드 구성">
    **요건**: 노드가 51% 이상 다른 금융기관 등으로 구성되어야 하며, 발행하려고 하는 증권 관련 사무를 처리에 적합하여야 함

    **Pulse 구현**:
    - 금융기관(증권사)이 Validator Node의 과반수 구성
    - 전자증권법상 계좌관리기관 참여
    - 증권 업무 처리에 적합한 인프라 구축
  </Accordion>
</AccordionGroup>

## 발행·유통 분리 원칙

### 이해상충 방지

금융위 가이드라인은 이해상충 방지를 위해 **발행과 유통(시장운영) 분리 원칙**을 적용합니다:

```
┌─────────────────────────────────────────────────────────┐
│                     분리 원칙                            │
│                                                         │
│  ┌──────────────┐          ┌──────────────┐            │
│  │    발행      │    ✕     │    유통      │            │
│  │  (A 증권사)   │──────────│  (A 증권사)   │            │
│  └──────────────┘          └──────────────┘            │
│                                                         │
│  발행·인수·주선한 증권은 해당 업자가 유통할 수 없음       │
└─────────────────────────────────────────────────────────┘
```

### 장외거래중개업자 역할

```
┌───────────┐     ┌────────────────────┐     ┌───────────┐
│   매수자   │────▶│ 장외거래중개업자   │◀────│   매도자   │
│           │주문  │  (C 증권사)        │주문  │           │
└───────────┘     │                    │     └───────────┘
                  │  다자간 상대매매   │
                  │  (호가 일치 시 체결) │
                  └────────────────────┘
```

**장외거래중개업자 규정:**
- 자사 고객 간 거래만 중개
- 다자간 상대매매 방식 (매수·매도 호가 일치 시 체결)
- 자기계약 금지

## 분산원장 포함 데이터

금융위 가이드라인에 따라 분산원장에 다음 데이터가 포함되어야 합니다:

| 데이터 항목 | 설명 | Pulse 구현 |
|-----------|------|-----------|
| **발행인** | 토큰 발행 주체 | Issuer 주소 |
| **계좌관리기관** | 권리 관리 금융기관 | Controller 지정 |
| **증권 종류** | 투자계약증권, 수익증권 등 | 메타데이터 |
| **종목 식별 정보** | 토큰 심볼, 이름 | name, symbol |
| **보유 수량** | 각 투자자의 보유량 | balanceOf |
| **발행 수량** | 총 발행량 | totalSupply |
| **질권 설정** | 담보 설정된 수량 | 파티션/Lock |
| **신탁재산 여부** | 신탁 관리 여부 | 메타데이터 |

## 법제화 전망

<Note>
  토큰증권 관련 법안(자본시장법, 전자증권법 개정)이 국회에서 통과되면, Pulse 네트워크는 추가 하위법령 요건에 맞춰 업그레이드될 예정입니다.
</Note>

### 예상 변경 사항

1. **분산원장 요건 상세화**: 하위법령에서 구체적 기술 요건 명시
2. **계좌관리기관 지정**: 공식 지정 절차 도입
3. **감독 체계**: 금융당국의 직접 모니터링 시스템 연동

Pulse는 Upgradable Smart Contract 구조를 채택하여 법제화 이후에도 유연하게 대응할 수 있습니다.

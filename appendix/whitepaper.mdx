## 백서 요약

### I. 서론

토큰증권(Security Token)은 기존 증권의 권리를 블록체인 토큰으로 표상한 것입니다. 2023년 2월 금융위원회의 '토큰증권 발행·유통 규율체계 정비방안' 발표 이후, 국내 토큰증권 시장이 본격적으로 형성되고 있습니다.

Pulse는 **신한투자증권 블록체인스크럼**과 **블록체인글로벌**이 공동 개발한 토큰증권 분산원장 플랫폼으로, 금융위 가이드라인을 완벽히 준수합니다.

### II. 인프라 아키텍처

### III. 블록체인 구조

Pulse는 **Hyperledger Besu** 기반의 프라이빗 블록체인입니다:

| 항목 | 사양 |
|------|------|
| **합의 알고리즘** | QBFT (Quorum BFT) |
| **블록 생성 주기** | 2초 |
| **트랜잭션 처리량** | 1,000+ TPS |
| **EVM 호환성** | 완전 호환 |

**노드 구성:**
- Validator Node: 신한투자증권, SK증권
- Boot Node: 네트워크 진입점
- RPC Node: API 요청 처리

### IV. 스마트 컨트랙트

Pulse는 **ERC-1400** 보안 토큰 표준을 채택했습니다:

```
┌─────────────────────────────────────────────────────────────┐
│                    ERC-1400 Umbrella                        │
│                                                             │
│  ┌───────────┐  ┌───────────┐  ┌───────────┐  ┌─────────┐ │
│  │ ERC-1410  │  │ ERC-1594  │  │ ERC-1643  │  │ERC-1644 │ │
│  │ Partition │  │  Core ST  │  │ Document  │  │Controller│ │
│  │  Token    │  │  Standard │  │Management │  │ Token   │ │
│  └───────────┘  └───────────┘  └───────────┘  └─────────┘ │
└─────────────────────────────────────────────────────────────┘
```

**업그레이드 가능한 컨트랙트:**
- Beacon Proxy 패턴 적용
- 무중단 업그레이드 지원
- 법제화 대응 유연성

### V. 키 관리 체계

3단계 키 관리 구조로 보안과 운영 효율성을 확보:

| 키 유형 | 역할 | 권한 |
|--------|------|------|
| **Issuer Key** | 발행사 | 토큰 발행/환수 |
| **Operator Key** | 계좌관리기관 | 일상 거래 처리 |
| **Controller Key** | 관리자 | 강제 전송, 동결 |

### VI. 발행/유통/총량관리 시스템

**발행 시스템:**
- 발행사가 토큰 생성 및 최초 발행
- 투자자에게 토큰 배정

**유통 시스템:**
- 장외거래중개업자를 통한 매매
- DvP 결제 (T+0)
- Atomic Swap으로 결제 리스크 제거

**총량관리 시스템:**
- 예탁결제원의 총량 모니터링
- 블록체인 직접 조회
- 실시간 감사 추적

### VII. 금융위 분산원장 요건 준수

Pulse는 금융위 가이드라인의 **8가지 분산원장 요건**을 모두 충족합니다:

<AccordionGroup>
  <Accordion title="1. 시간순 기록 및 변경 방지">
    블록체인 특성상 모든 트랜잭션은 타임스탬프와 함께 순차 기록되며, 블록 해시 체인으로 사후 변경이 불가능합니다.
  </Accordion>
  <Accordion title="2. 동일성 입증 가능">
    계좌관리기관이 Operator Key로 모든 거래에 서명하며, 트랜잭션 해시로 거래를 추적할 수 있습니다.
  </Accordion>
  <Accordion title="3. 복수 분산 장부 기록">
    다수의 Validator Node가 QBFT 합의를 통해 동일한 원장을 유지합니다.
  </Accordion>
  <Accordion title="4. 독립 계좌관리기관 참여">
    신한투자증권, SK증권 등 발행인과 특수관계 없는 금융기관이 노드로 참여합니다.
  </Accordion>
  <Accordion title="5. 별도 가상자산 불필요">
    프라이빗 네트워크로 Gas Fee를 부과하지 않으며, 별도 코인/토큰이 필요 없습니다.
  </Accordion>
  <Accordion title="6. 적합한 권리 등록">
    투자계약증권, 수익증권, 채무증권을 지원하며, 상장증권 등은 배제합니다.
  </Accordion>
  <Accordion title="7. 법령 준수">
    개인정보는 분산원장에 직접 저장하지 않고, 지갑 주소와 분리 관리합니다.
  </Accordion>
  <Accordion title="8. 금융기관 노드 구성">
    금융기관(증권사)이 Validator Node의 과반수를 구성합니다.
  </Accordion>
</AccordionGroup>
